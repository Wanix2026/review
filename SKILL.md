# 复习资料整理

以大纲为骨架、PPT为主要内容源，生成结构化复习笔记。

## 铁律

1. 每条知识点必须能追溯到大纲/PPT原文 — 找不到来源就标注 `[待补充]`
2. 数值原文照录，不改格式、不丢单位、不四舍五入（`5-10 cmH₂O` ≠ `5~10`）
3. 操作步骤逐条列出不合并，表格保持结构不拆散，**穷举提取不归纳总结**
4. 大纲子项全覆盖 — 大纲写6个子项输出必须有6个，缺一漏一
5. PPT没写的内容标注 `[PPT缺失]`，**绝不**自己推断补全，哪怕答案看起来"显然"
6. **绝不**用"等"省略条目，**绝不**跨文件拼接知识点，**绝不**凭记忆写摘要，**绝不**改写原文使之"更流畅"
7. **绝不**把多个独立知识点压进一个段落——每个大纲子项独立成 `###` 小节

## 流程

### Phase 0 · 准备（用户确认一次后全自动）

**Step 1 — 扫描 + 分类**
列出目录下所有文件。按文件名+内容特征自动分五类：

| 类型 | 关键词 | 内容特征 |
|------|--------|---------|
| 大纲 | 大纲/syllabus/课程标准/课程简介 | "掌握/熟悉/了解"+章节结构 |
| 教学日历 | 日历/进度表/授课计划/课程安排 | "周次/日期/学时" |
| PPT | .pptx/.ppt/.pdf(课件目录) | 标题+列表/表格 |
| 复习重点 | 复习重点/考试重点/考点/题型 | 题型/分值分布 |
| 题库 | 题库/试题/习题/考试宝 | 题目编号+选项+答案 |

分类后展示摘要。大纲和日历必须存在（日历可能嵌入大纲末页）。

**Step 2 — 章节映射**
建立大纲章节→日历周次→PPT文件的对应关系，四层匹配：
1. 编号（"第三章"↔"Week5"↔"03-xxx"）
2. 内容采样（打开PPT首slide实际标题 → 提取关键词 → 与大纲每章"教学内容"段做子串重叠比对 → 重叠度最高的匹配。关键：PPT用教材章号而大纲用自编号时，这步是唯一正确匹配方式）
3. 关键词重叠度
4. 日历桥接

置信度≥85%自动通过，<50%标注需确认。展示映射表。

**Step 3 — 上下文确认**
- 从大纲"考核方式"段自动抽取：考试课/考查课、卷面门槛
- 从复习重点文件自动扫描题型分布
- 检测不到题型信息 → 默认纯知识点整理，不追问
- 第二次使用该目录 → 读取 `.course-cache/config.json` 跳过全部确认
- 维护 `~/.claude/skills/review/projects.json` 记录所有处理过的课程及其配置

**Step 4 — 环境检查**
检查 pdf2txt/python-pptx/python-docx/openpyxl 是否可用，缺失提示安装命令。

如果检测到 `.course-cache/manifest.json`，询问：更新模式（增量）还是完整重建。

---

### Phase 1 · 提取

运行 `python3 scripts/extract.py .course-cache/extracted/ @filelist.txt` 批量提取。

**规则**：
- 每个文件计算 SHA256，已完成的不重复提取
- PDF先诊断（文字版/扫描版/加密），失败给具体修复建议：
  - 加密 → "请用Preview打开，File→Export→取消密码后重新放入"
  - .doc → "请用Word打开后另存为.docx"
  - 扫描版 → "自动切换OCR，如效果不好建议用Adobe Scan重扫"
- PPT提取时标注 SmartArt/图片缺失
- 进度输出 `[████░░] 8/13`
- 提取后运行 `python3 scripts/verify.py .course-cache/manifest.json .course-cache/extracted/`
- 不合格文件自动降级重试1次

---

### Phase 2 · 验证

从缓存读取三源，逐章对照决策表：

| 大纲 | 日历 | PPT | 判定 |
|------|------|-----|------|
| 掌握 | 有 | 有 | 确认 |
| 掌握 | 有 | 部分 | 部分覆盖 |
| 掌握 | 有 | 无 | 致命缺口 |
| 掌握 | 无 | 有/无 | 不一致/存疑 |
| 熟悉 | 有 | 无 | 警告缺口 |
| 无 | 有 | 有 | 新增（大纲未更新） |

冲突处理：
1. 日历与大纲冲突 → 以日历为准，保留大纲原文为注释
2. PPT与大纲冲突 → 以PPT为准
3. 两份PPT数值冲突 → `[两版PPT不一致: A写X, B写Y，建议核实]`
4. PPT内部前后矛盾 → `[PPT此处写X，但前文写Y，以最新PPT为准]`
5. 大纲说"了解"但日历给大量学时 → `[大纲与实际教学不符]` 建议升级权重
6. 所有冲突显式标注，不静默裁决

有致命缺口时展示缺口报告，每项缺三选一：补文件 / 确认缺失 / 移除此章。最多2轮。

---

### Phase 3 · 生成

**关键：逐章生成，不可一口气写完所有章节。每章完成后自查，通过才进入下一章。**

#### Step 3.0 · 逐章前准备

列出大纲全部章节 + 对应 PPT 文件 + PPT 总 slide 数。这步确认"要写多少"。

#### Step 3.1 · 逐章生成循环

对每一章，按以下流程执行：

```
① 打开该章对应的 PPT 缓存文件，读完全部内容
② 列出该 PPT 包含的所有知识点条目（不遗漏任何一个 slide）
③ 按模板写出该章内容
④ 自查：
   - 大纲子项是否全部出现？
   - PPT 每个 slide 的内容是否都有体现？
   - 数值是否都加了 **？
   - 对比内容是否用了表格？
   - 该章 h3 数量是否 ≥ 大纲子项数？
⑤ 自查不通过 → 回到②补全，不通过不进入下一章
⑥ 自查通过 → 进入下一章
```

每章按此模板输出：

```markdown
## 第X章 章节名
📄 来源: 文件名

### 大纲要求
● 掌握: ...（大纲原文引用）
○ 熟悉: ...
· 了解: ...

### {知识点1}（按大纲子项逐个设为h3小节，不合并）
[一个h3小节只说一个知识点]
[对比类→表格 | 数值类→表格 | 步骤类→逐条列出 | 概念类→**粗体**定义句]

### {知识点2}
...

### {知识点N}

### 可能的考点
PPT末页思考题 → [原文照录]
大纲双重点 → [重点∩难点]
课堂测验 → [日历标记]
```

**强制格式规则**：

1. **穷举提取，禁止归纳**：PPT 每一页的每个知识条目（数值、定义、步骤、分类、对比、临床举例）必须出现在输出中。输出文字量不应低于 PPT 原文量的 **60%**。宁可多不可少
2. **小节拆分**：大纲每个子项独立为 `###` 小节，不同知识点绝不压进同一个 h3
3. **表格优先**：对比内容、数值参数、分类体系 → **必须**用表格，不用段落
4. **数值全加粗**：每个精确数值用 `**数值**` 加粗标记
5. **术语全加粗**：每个医学核心术语首次出现时用 `**术语**` 加粗
6. **步骤逐条列**：操作步骤、处理原则用 `1. 2. 3.` 逐条列出
7. **章节标题不写学时**：学时信息属于教学管理，不体现在复习资料章节标题中。章节标题只保留章序号和名称。重点章（多学时+多重点）在标题后加 `重点章` pill 标记
8. **章节末尾加权重标注**：`大纲重点Y项 · 难点Z项`
9. **生成后自检**：输出后对比该章 PPT 原文——如果 PPT 有 2000 字而输出只有 500 字，说明提取不充分，必须补全重写
10. **课程管理信息后置**：成绩评定方案、在线课程平台、考核占比、登录方式等课程行政信息 → 放在文档末尾（审查报告之前），**不要**混入任何章节的内容中。头部 blockquote 只保留来源说明和教材信息

**全部章节写完后自检**（不可跳过）：
- 大纲子项覆盖率≥90%
- 抽查5个数值与PPT原文比对
- 搜索"等""主要""大致""约""一般"确认无违规使用
- **反向验证**：统计输出总字符数 / PPT原文总字符数。比值 < 50% → 说明某章提取不充分，回到该章的②重新列出知识条目、补全重写。**最多迭代 2 轮**，2 轮后仍不达标则在审查报告中注明
- 审查报告附在笔记末尾：

```markdown
## 审查报告
| 检查项 | 结果 |
|--------|------|
| 大纲子项覆盖率 | X/Y · 缺Z项已标注 |
| 数值抽查 | 5/5 一致 |
| 源文件追溯 | 完整 |
| 禁止项 | 0处 |
```

### 衍生输出（生成后自动）

- **Anki**: `python3 scripts/anki.py <笔记.md>` → CSV记忆卡
- **Mermaid**: `python3 scripts/mindmap.py <笔记.md>` 嵌入 Markdown

**Phase 3 末尾自动执行**：生成完 MD 后立即运行 `python3 scripts/to_html.py <笔记.md>` 生成 HTML。如果转换脚本报错或内容比<50%，检查 MD 格式后重新生成。

---

### 题库处理（可选）

题库文件存在时：
- Web平台（人卫/学习通）→ 浏览器Console JS提取页面文本 → 解析
- PDF题库 → pdf2txt → 解析
- 清洗保留：题型+题干+选项+正确答案
- 删除：解析+学生答案+分隔符
- 输出 Word (.docx)，题目间空行分隔

---

## HTML 输出

运行 `python3 scripts/to_html.py <笔记.md>` 自动生成 HTML。模板为 `references/template.html`，替换占位符：
- `{{COURSE_NAME}}` → 课程名
- `{{SIDEBAR_ITEMS}}` → 侧栏链接（`<li><a href="#sec-xxx">章节名</a></li>`）
- `{{CONTENT}}` → 正文HTML
- `{{GAP_SUMMARY}}` → 缺口计数

生成后同时输出 Markdown 和 HTML。PDF 可选。
