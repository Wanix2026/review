# 复习资料整理技能

为医学生系统整理课程复习资料。从分散的源文件（大纲、教学日历、PPT/PDF课件、复习重点、题库）中提取知识点，以大纲为骨架、教学日历为时间轴、PPT为主要内容源，生成结构化复习笔记。

## 核心原则

1. **大纲是唯一的真理源** — 所有输出内容必须能回溯到大纲中的某一行
2. **三源交叉验证** — 大纲、教学日历、PPT 三者交叉比对，不一致处显式标注
3. **日历当学期为准** — 大纲与日历冲突时以日历为准，但保留证据
4. **PPT内容优先** — 大纲与PPT内容冲突时以PPT为准（最新授课内容）
5. **不生成内容** — 只整理、提取、归类，不凭空编写知识点
6. **缺口显式标注** — 资料缺失处明确标记，不与正常内容混淆

## 执行流程

### Phase 0 · 文件发现与分类

**目标**: 识别所有源文件，确认每份文件的角色，建立章节映射。

#### Step 0.1 · 扫描目录

扫描用户指定的课程目录，列出所有文件（含子目录）。

#### Step 0.2 · 自动分类

按文件名关键词 + 内容抽样将文件分为五类：

| 类型 | 文件名关键词 | 内容特征 |
|------|-------------|---------|
| **大纲** | 大纲、syllabus、课程标准、课程简介、教学纲要、教学基本要求 | "掌握/熟悉/了解"、章节结构、考核目标 |
| **教学日历** | 日历、进度表、教学计划、授课计划、课程安排、周历 | "周次/日期/学时"、授课教师 |
| **PPT课件** | .pptx、.ppt、课件中常见的章节标题 | 标题+列表/表格结构 |
| **复习重点** | 复习重点、考试重点、重点难点、复习提纲、考点、题型 | 题型/分值/考试范围 |
| **题库** | 题库、试题、习题、考试宝 | 题目编号+选项+答案 |

#### Step 0.3 · 分类验证

对每份分类结果做内容抽样验证：
- 分类为"大纲" → 检查是否含"掌握/熟悉/了解" → 无则告警："此文件可能不是大纲，确认为教学日历？"
- 分类为"教学日历" → 检查是否含"周次/日期/学时" → 无则告警
- 发现多份大纲 → 相似度对比 → 提示用户选择当学期版本

#### Step 0.4 · 重复检测

同章节文件名相似度 > 80% 或首页标题相同的文件 → 标记为重复组 → 保留最新修改时间 → 旧版本归档。

#### Step 0.5 · 前置闸门

**必须确认**以下文件存在后才继续：
- 🔒 **教学大纲** — 定义考试范围和重难点层级
- 🔒 **教学日历** — 验证课时完整性 + 学时分→考频权重

如果缺失：明确告知用户缺失什么、为什么必要、提供什么替代方案。

日历可能内嵌在大纲末页——检查大纲最后几页是否有课时安排表。

#### Step 0.6 · 章节映射

建立三源（大纲、日历、PPT）之间的章节对应关系。

**四层匹配策略**（前两层自动，后两层人工）：
1. **编号匹配**: 大纲"第三章" ↔ 日历"Week5→第三章" ↔ PPT文件名含"3/三/03"
2. **内容采样匹配**: 打开PPT第一页实际标题，提取章节关键词 → 与大纲每章"教学内容"段落做关键词重叠比对 → 重叠度最高的大纲章节即为匹配。**这步解决PPT用教材章号、大纲用自编章号的常见不一致**
3. **关键词匹配**: 公共子串/关键词重叠度 ≥50% → 自动匹配，<50% → 列入未匹配清单
4. **日历桥接**: 当 PPT 文件名与大纲差异极大时，用日历的周次作为桥（同一天讲的课，大纲章节和PPT必然对应）

展示映射表请用户确认：

```
| 大纲章节 | 日历周次 | 匹配PPT | 置信度 |
|----------|---------|---------|:--:|
| Ch3 外科感染 | Week5: 外科感染 | 感染.pdf | 90% |
| Ch4 创伤 | Week6: 创伤与急救 | ⚠️ 未找到 | 0% |
```

#### Step 0.7 · 医学上下文确认

- 确认考试课/考查课
- **如果用户主动提供了题型分布**，记录（A1/A2/B1/简答/判断/名词解释及各自分值）
- **如果用户未提供题型**：不追问，进入纯知识点整理模式

#### Step 0.8 · 环境依赖预检

在用户确认分类后、开始提取前，检查必需工具：

```
□ pdf2txt      → which pdf2txt 或 pip3 show pdfminer.six
□ python-pptx  → python3 -c "from pptx import Presentation"
□ python-docx  → python3 -c "from docx import Document"
□ openpyxl     → python3 -c "import openpyxl"
```

缺失的工具 → 立即输出安装命令（`pip3 install xxx`），等用户安装后再继续。
pdf2txt 不可用时检查 `pip3 install pdfminer.six`。

### 快速更新模式

如果课程目录下已存在 `.course-cache/manifest.json`，说明之前已生成过复习资料。此时询问用户：

> "检测到已有缓存（上次提取时间: xxx）。是否进入更新模式？"
> - [1] 更新模式：只扫描新文件/变化文件 → 增量提取 → 仅更新受影响章节
> - [2] 完整重建：忽略缓存，从头处理所有文件

更新模式下：
- 扫描目录 → 对比 manifest.json 中的文件指纹
- 新文件/指纹变化的文件 → 增量提取 → 更新缓存
- 删除的文件 → manifest 中标记 removed
- 仅重新生成受影响章节，其余章节保留不变

### Phase 1 · 全量提取

**目标**: 一次性提取所有文件的文本内容，写入缓存，不再重复提取。

#### Step 1.1 · 建立缓存

在课程目录下创建 `.course-cache/`：
```
.course-cache/
├── manifest.json     ← 文件指纹注册表
├── extracted/        ← 提取出的纯文本
├── gap-report.md     ← 缺口报告
└── mapping.json      ← 章节映射结果
```

#### Step 1.2 · 文件指纹

每个源文件计算 SHA256，存入 manifest.json：
```json
{
  "大纲.pdf": {"sha256": "abc123", "status": "ok", "pages": 12},
  "休克.pdf": {"sha256": "def456", "status": "ok", "pages": 62},
  "感染.pdf": {"sha256": "ghi789", "status": "encrypted", "note": "需解密"}
}
```

#### Step 1.3 · 格式诊断与工具选择

| 格式 | 诊断方式 | 主力工具 | 兜底方案 |
|------|---------|---------|---------|
| PDF(文字版) | pdf2txt 首1KB > 100字符 | `pdf2txt -o out.txt file.pdf` | — |
| PDF(扫描版) | pdf2txt 首1KB < 100字符 | macOS Vision OCR | tesseract |
| PDF(加密) | pdf2txt 报 encrypted | 跳过，提示用户解密 | — |
| .docx | 直接读取 | python-docx | — |
| .doc | 不可用 python-docx | `textutil -convert txt` | antiword |
| .pptx | 直接读取 | python-pptx | — |
| .xlsx | 直接读取 | openpyxl | — |

#### Step 1.4 · 批量提取

- 每个文件提取全文，写入 `extracted/<文件名>.txt`
- 提取失败的记录到 manifest.json 并分类（加密/损坏/扫描件/空文件）
- PPT提取时标注不可提取项：`⚠️ Slide 12: [SmartArt流程图] [图片]`

#### Step 1.5 · 完整性自检

提取后立即自动检查：
- PDF: 提取文本页数 = 原始PDF页数？
- PPT: 提取Slide数 = PPT总Slide数？
- 文字密度: 平均每页 > 100字符？
- 乱码检测: 乱码字符占比 < 5%？

不合格 → 自动降级重试（如 pdf2txt → OCR），最多重试 1 次。

---

### Phase 2 · 三源交叉比对

**目标**: 逐章比对大纲、教学日历、PPT，生成带置信度的章节树。

#### Step 2.1 · 三源提取

从缓存中读取三类源文件，逐章建立索引：
- 大纲 → 章节结构 + 掌握度（掌握/熟悉/了解）+ 重难点标注 + 子项列表
- 教学日历 → 周次 → 章节对应 + 学时数
- PPT → 章节归属 + 内容摘要

#### Step 2.2 · 16状态决策表

对每个章节，三源各取状态（大纲: 掌握/熟悉/了解/无，日历: 有/无，PPT: 有/部分/无），对照判定：

| 大纲 | 日历 | PPT | 判定 |
|------|------|-----|------|
| 掌握 | 有 | 有 | ✅ 确认 |
| 掌握 | 有 | 部分 | 🟡 部分覆盖 · 标注缺失子项 |
| 掌握 | 有 | 无 | 🔴 致命缺口 · 应教但缺PPT |
| 掌握 | 无 | 有 | ⚠️ 不一致 · 大纲重点+有PPT但日历未排 |
| 掌握 | 无 | 无 | 🟡 存疑 · 大纲写但本学期可能已删 |
| 熟悉 | 有 | 有 | ✅ 确认 |
| 熟悉 | 有 | 无 | 🟡 警告缺口 |
| 熟悉 | 无 | * | 🔵 降级补充 |
| 了解 | * | * | 🔵 次要内容 |
| 无 | 有 | 有 | 🟡 新增 · 日历+PPT均有，大纲未更新 |
| 无 | 有 | 无 | 🔵 提示 |
| 无 | 无 | 有 | 🔵 额外资料 |

**冲突处理规则**:
1. 日历与大纲冲突 → 以日历为准（当学期真实安排）
2. PPT与大纲冲突 → 以PPT为准（最新授课内容）
3. 大纲说"了解"但日历给6学时+PPT184页 → ⚠️ [大纲标记与实际教学不符]，建议升级为掌握
4. 所有冲突都显式标注，保留证据，不静默裁决

#### Step 2.3 · 缺口反馈

有 🔴 致命缺口时 → 展示缺口报告 → 每项缺口支持三种响应：
- `[1] 补充文件` → Phase 1增量提取 → 重新比对
- `[2] 确认缺失` → 缺口标记降级，标注"已知缺失"
- `[3] 移除此章` → 从章节树中移除

最多 2 轮反馈后强制进入 Phase 3。

### Phase 3 · 复习资料生成

**目标**: 以大纲为骨架，逐章填充知识点，生成 Markdown + HTML + 可选 PDF。

**双模式行为**：

| 条件 | 行为 |
|------|------|
| **用户提供了题型分布** | 按题型驱动格式（名词解释/A1/A2/B1/简答/判断），启用 Step 3.2-3.3 |
| **用户未提供题型**（默认） | 纯知识点整理模式：不区分题型、不输出名词解释专栏、不用任何题型标签 |

#### Step 3.1 · 知识点提取与组织

对每个章节，按大纲子项逐条整理。**必须完整读取 PPT 全文后整理，不可凭记忆写摘要**。

```
大纲: "掌握休克的概念、分类、临床表现、诊断和治疗原则"
  ├─ 休克的概念 → 完整定义
  ├─ 休克的分类 → 对比表（5型休克的机制/特征/代表疾病）
  ├─ 临床表现 → 分期+各期表现
  ├─ 诊断 → 诊断标准+监测指标
  └─ 治疗原则 → 分点条目（补液/血管活性药/病因）
```

**内容深度要求**：
- PPT中的精确数值（温度、时间、剂量、正常值范围）→ 必须原文照录
- PPT中的操作步骤/处理原则 → 逐条列出，不合并不简化
- PPT中的对比表格 → 保持表格结构，不转为散装文字
- PPT缺失的内容 → 从大纲中提取子项标题，标注 [PPT缺失]

**信息保真度优先级**（有题型时适用）: 名词解释 > 简答 > A2 > B1 > A1 > 判断
高优先级格式一旦建立即冻结，低优先级引用时不得缩减。

#### Step 3.2 · 名词解释提取（仅在用户提供了题型时执行）

**触发条件**: Phase 0 确认了考试题型包含"名词解释"。

**提取流程**:
- Step 0: 先检查大纲中是否有独立概念清单 section（"基本概念""术语表""名词解释"）→ 最高优先级
- Step 1: 匹配大纲中 5 种表述模式:
  - 明示型: "掌握...的概念/定义"
  - 领会型: "掌握[医学术语]"（无"概念"二字但术语本身是概念）
  - 对比型: "掌握X与Y的区别"（X和Y各是名词解释）
  - 嵌套型: "掌握X及其临床应用"（X是概念）
  - 隐含型: "了解[术语]"
- Step 2: 交叉验证（PPT标题页/概念框 → 提升置信度）
- Step 3: 去噪（排除"...的方法/步骤/技术操作"，保留"疾病名/综合征名/体征名/评分系统名"）

**输出格式**: 
- 完整定义，不缩减（记忆型题目，必须完整答出）
- 按掌握度着色：🔴掌握=红色文字 | 🟡熟悉=橙琥珀色 | 🔵了解=灰色
- 附章节归属: "(第X章)"

#### Step 3.3 · 题型驱动格式选择（仅在用户提供了题型时执行）

**触发条件**: Phase 0 确认了具体题型分布。

| 题型 | 大纲信号词 | 输出格式 |
|------|-----------|---------|
| 名词解释 | 概念、定义、含义、概述 | 概念名 + 完整定义(4-6要点) |
| A1型 | 正常值、标准、分期、首选、金标准 | 精确数值/标准表述，红色高亮数值 |
| A2型 | 病因、临表、诊断、治疗 | 因果链: 病因→病理→临表→检查→诊断→治疗 |
| B1型 | 区别、分类、特点、鉴别 | 横向对比表，维度一致并列 |
| 简答题 | 原则、方法、流程、措施、>3子项 | 分点条目，保持PPT原文顺序 |
| 判断题 | 无直接信号 | 从数值/绝对化表述自动衍生，标注[衍生题] |

**因果链按学科适配**:
- 临床学科: 病因→病理生理→临床表现→检查→诊断→治疗
- 预防医学: 暴露因素→健康效应→疾病分布→预防策略
- 诊断学科: 临床表现→检查方法→结果判读→诊断→鉴别

#### Step 3.4 · 数值标注

医学考试的A1型大量考察精确数值。提取时自动标注：

- 🔴 必背数值: 正常值范围、分期标准、药物剂量、时间窗口
- 🟡 参考范围: 实验室检查参考区间
- 🔵 对比数值: 成人vs儿童、生理vs病理

#### Step 3.5 · 章节模板（每章固定三段式）

```markdown
## 第X章 章节名（X学时）

📄 来源: 01-课件名.pdf

### ● 大纲要求
● 掌握: ...（直接引用大纲重点原文）
○ 熟悉: ...（直接引用大纲难点原文）
· 了解: ...（大纲标注为了解的内容）

### 内容要点
[从PPT提取的知识点，按大纲子项逐条组织]
[数值、标准、时间窗等精确参数用表格呈现]
[操作步骤/原则逐条列出]
[缺失处标注 [PPT缺失]]

### 可能的考点
PPT思考题 → [PPT原文中的思考题/案例分析题，原文照录]
大纲双重点 → [大纲同时标注"重点"+"难点"的知识点——几乎必考]
学时信号 → [同章节多学时但大纲标注为"了解"——实际教学可能加重]
```

**用户提供题型时**，内容要点中可以加微型题型标签（名词/数值/对比/因果/简答）。
**用户未提供题型时**，不输出任何题型标签，"可能的考点"基于 PPT 思考题和大纲重点推断，不硬套题型。

#### Step 3.6 · 缺口标注

```markdown
┃ 🔴 资料缺失 · 待补充
┃ 本章大纲标注为"掌握"，但未找到对应授课PPT。
┃ 以下仅为大纲要求的知识点框架。
```

### Phase 3 附加 · 题库处理（可选）

当用户提供题库文件时执行。

#### 题库提取

| 来源 | 提取方式 |
|------|---------|
| Web平台（人卫/学习通） | 浏览器Console执行JS提取页面文本 → 解析 |
| PDF题库 | pdf2txt提取文本 → 解析 |
| Word/文本 | 直接读取解析 |

#### 清洗规则

```
保留: 题型、题干、选项、正确答案
删除: 试题解析、学生答案、分隔符（破折号/横线）、来源标注
检查: 选项去重（学习通常见问题）
```

#### 输出格式

- **Word (.docx)** — 用于导入考试宝等刷题软件
- 题目间空行分隔，不额外标注来源

#### 答案补全

如题库只有题目无答案 → AI可基于医学知识推理补全
→ 标注 `[AI推断]` 与 `[原文答案]` 区分
→ 补全后用户检查确认

#### 知识点重合度

可选：将题库题目按关键词归类到大纲章节，统计每章题量 → 与复习资料的考点权重交叉对比 → 输出重合度报告。

## HTML 输出规范

### 设计原则

- 极简配色：白底黑字，红琥珀灰三层着色
- 排版层级区分结构，不使用装饰性 emoji
- 侧栏目录 + 滚动监听（IntersectionObserver）
- 移动端 off-canvas 侧栏 + 打印样式

### 视觉体系

**三层信号**:

| 层级 | 用途 | 方式 |
|------|------|------|
| 结构类 | 章节/大纲/预测标记 | 文字 pill 标签，灰色背景 |
| 优先级类 | 掌握度区分 | ● ○ · 圆点字符 + 文字颜色 |
| 警示类 | 缺口/冲突标注 | 彩色左边框（红/琥珀/蓝） |

**颜色参数**:

```
背景:      #fff (纯白)
正文:      #2c2c2c
掌握(red): #c41e3a bold
熟悉(amber): #b45309
```

## 交互原则

1. **Phase 0 必须用户确认**：文件分类、章节映射、考试课/考查课 — 确认后再继续。题型分布由用户主动提供，不追问
2. **Phase 1 完全自动化**：提取过程不打断用户
3. **Phase 2 仅在发现致命缺口时打断**：最多 2 轮
4. **Phase 3 直接输出结果**：不中途追问
5. **默认纯知识点整理**：用户未提供题型时，不区分题型、不输出名词解释专栏、不用题型标签

## 参考资料

本技能的设计基于真实的临床医学期末复习经验，针对以下痛点优化：
- PDF提取不可靠（Read tool不稳定 + pdf2txt碎片化使用）
- 大纲关键词匹配不完整导致名词解释多次返工
- 源文件缺失导致复习资料可靠度下降
- HTML排版迭代多轮才找到清爽+重点突出+目录跳转的平衡点

详细的题型提取策略、视觉设计规范、三源交叉决策表见上文各 Phase 说明。
