# 复习资料整理

以大纲为骨架、PPT为主要内容源，生成结构化复习笔记。

## 铁律

1. 每条知识点必须能追溯到大纲/PPT原文 — 找不到来源就标注 `[待补充]`
2. 数值原文照录，不改格式、不丢单位（`5-10 cmH₂O` ≠ `5~10`）
3. 操作步骤逐条列出不合并，表格保持结构不拆散
4. 大纲子项全覆盖 — 6个子项缺一个就是遗漏
5. PPT没写的内容标注 `[PPT缺失]`，不自己推断
6. 不用"等"字省略，不跨文件拼接知识点，不凭记忆写摘要

## 流程

### Phase 0 · 准备（用户确认一次后全自动）

**Step 1 — 扫描 + 分类**
列出目录下所有文件。按文件名+内容特征自动分五类：

| 类型 | 关键词 | 内容特征 |
|------|--------|---------|
| 大纲 | 大纲/syllabus/课程标准/课程简介 | "掌握/熟悉/了解"+章节结构 |
| 教学日历 | 日历/进度表/授课计划/课程安排 | "周次/日期/学时" |
| PPT | .pptx/.ppt/.pdf(课件目录) | 标题+列表/表格 |
| 复习重点 | 复习重点/考试重点/考点/题型 | 题型/分值分布 |
| 题库 | 题库/试题/习题/考试宝 | 题目编号+选项+答案 |

分类后展示摘要。大纲和日历必须存在（日历可能嵌入大纲末页）。

**Step 2 — 章节映射**
建立大纲章节→日历周次→PPT文件的对应关系，四层匹配：
1. 编号（"第三章"↔"Week5"↔"03-xxx"）
2. 内容采样（打开PPT第一页标题 vs 大纲章节的内容描述 → 解决PPT用教材章号的常见不一致）
3. 关键词重叠度
4. 日历桥接

置信度≥85%自动通过，<50%标注需确认。展示映射表。

**Step 3 — 上下文确认**
- 从大纲"考核方式"段自动抽取：考试课/考查课、卷面门槛
- 从复习重点文件自动扫描题型分布
- 检测不到题型信息 → 默认纯知识点整理，不追问
- 第二次使用该目录 → 读取 `.course-cache/config.json` 跳过全部确认

**Step 4 — 环境检查**
检查 pdf2txt/python-pptx/python-docx/openpyxl 是否可用，缺失提示安装命令。

如果检测到 `.course-cache/manifest.json`，询问：更新模式（增量）还是完整重建。

---

### Phase 1 · 提取

运行 `python3 scripts/extract.py .course-cache/extracted/ @filelist.txt` 批量提取。

**规则**：
- 每个文件计算 SHA256，已完成的不重复提取
- PDF先诊断（文字版/扫描版/加密），加密PDF给修复建议
- PPT提取时标注 SmartArt/图片缺失
- 进度输出 `[████░░] 8/13`
- 提取后运行 `python3 scripts/verify.py .course-cache/manifest.json .course-cache/extracted/`
- 不合格文件自动降级重试1次

---

### Phase 2 · 验证

从缓存读取三源，逐章对照决策表：

| 大纲 | 日历 | PPT | 判定 |
|------|------|-----|------|
| 掌握 | 有 | 有 | 确认 |
| 掌握 | 有 | 部分 | 部分覆盖 |
| 掌握 | 有 | 无 | 致命缺口 |
| 掌握 | 无 | 有/无 | 不一致/存疑 |
| 熟悉 | 有 | 无 | 警告缺口 |
| 无 | 有 | 有 | 新增（大纲未更新） |

冲突处理：日历优先、PPT内容优先、两份PPT数值冲突→标注 `[PPT-A写X, PPT-B写Y]`

有致命缺口时展示缺口报告，每项缺三选一：补文件 / 确认缺失 / 移除此章。最多2轮。

---

### Phase 3 · 生成

对每个章节按固定模板输出：

```markdown
## 第X章 章节名（X学时）
📄 来源: 文件名

### 大纲要求
● 掌握: ...
○ 熟悉: ...
· 了解: ...

### 内容要点
[PPT知识点，按大纲子项逐条组织]
[数值用表格，步骤逐条列，缺失处标[PPT缺失]]

### 可能的考点
PPT末页思考题 → [原文照录，搜索末页含"思考/复习/?/question"的slide]
大纲双重点 → [重点∩难点]
课堂测验 → [日历中该章的阶段性测验标记]
```

**内容要求**：每章必须读完PPT全文后落笔。数值原样照录。不用题型标签（除非用户提供了题型分布）。

**生成后自检**（不可跳过）：
- 大纲子项覆盖率≥90%
- 抽查5个数值与PPT原文比对
- 搜索"等""主要""大致""约""一般"确认无违规使用
- 审查报告附在笔记末尾：

```markdown
## 审查报告
| 检查项 | 结果 |
|--------|------|
| 大纲子项覆盖率 | X/Y · 缺Z项已标注 |
| 数值抽查 | 5/5 一致 |
| 源文件追溯 | 完整 |
| 禁止项 | 0处 |
```

---

### 衍生输出（生成后自动）

- **Anki**: `python3 scripts/anki.py <笔记.md>` → CSV记忆卡
- **Mermaid**: `python3 scripts/mindmap.py <笔记.md>` 嵌入 Markdown

---

### 题库处理（可选）

题库文件存在时：
- Web平台（人卫/学习通）→ 浏览器Console JS提取页面文本 → 解析
- PDF题库 → pdf2txt → 解析
- 清洗保留：题型+题干+选项+正确答案
- 删除：解析+学生答案+分隔符
- 输出 Word (.docx)，题目间空行分隔

---

## HTML 输出

使用 `references/template.html`，替换三处占位符：
- `{{COURSE_NAME}}` → 课程名
- `{{SIDEBAR_ITEMS}}` → 侧栏链接（`<li><a href="#sec-xxx">章节名</a></li>`）
- `{{CONTENT}}` → 正文HTML
- `{{GAP_SUMMARY}}` → 缺口计数

生成后同时输出 Markdown 和 HTML。PDF 可选。
